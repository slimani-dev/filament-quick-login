# Contributing

Thank you for considering contributing to the Quick Login!

## Code of Conduct

Please review and adhere to our Code of Conduct.

## Security Vulnerabilities

If you discover a security vulnerability within Quick Login, please send an e-mail to security@slimani.dev. All security vulnerabilities will be promptly addressed.

## Pull Requests

1. Fork the repository and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Coding Standards

The Quick Login follows the PSR-1, PSR-2, and PSR-4 coding standards.
